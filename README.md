# grabdata.dev — сайт-визитка Аллы Станчевой (статическая версия)

Одностраничный сайт-визитка (Junior Data Analyst, Amsterdam). Чистая статика —
HTML + CSS, без сборки и без фреймворков. Живёт на собственном сервере (Hetzner)
под доменом **grabdata.dev**, отдаётся через nginx с HTTPS-сертификатом Let's Encrypt.

## Что где лежит

| Файл | Что это |
|------|---------|
| `index.html` | весь сайт: тексты, секции, структура. **Тексты правишь здесь.** |
| `styles.css` | оформление. Цвета и шрифт — вверху, в блоке `:root`. |
| `images/cover.jpg` | фото в Hero. Заменить — положи новый файл с тем же именем. |
| `favicon.svg` | иконка вкладки браузера. |

## Как править сайт самой

1. **Меняешь текст** — открываешь `index.html`, находишь нужный кусок (секции подписаны
   комментариями `<!-- PROJECTS -->`, `<!-- ABOUT -->` и т.д.), правишь, сохраняешь.
2. **Меняешь цвет акцента** — `styles.css`, блок `:root`, строка `--color-accent`.
3. **Проверяешь локально** — открываешь `index.html` в браузере двойным кликом
   (или через Live Server в VS Code).
4. **Публикуешь** — три шага:
   ```
   git add .
   git commit -m "Что изменила"
   git push
   ```
   Затем просишь агента: «@sysadmin обнови сайт grabdata.dev — забери последнюю версию из репозитория».
   Агент сделает `git pull` на сервере, и через несколько секунд правка в проде.

## Откат, если что-то сломала

```
git revert HEAD
git push
```
И снова просишь `@sysadmin` обновить сайт. Прошлая рабочая версия вернётся.

## Связь с проектом-исходником

Дизайн взят из Astro-проекта `сайт_визитка_Алла_Станчева_2` (соседняя папка).
Эта версия — самостоятельная статическая копия для боевого хостинга на своём сервере.
Если меняешь дизайн в Astro-исходнике — перенеси правки сюда вручную (версии независимы).
